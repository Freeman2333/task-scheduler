## Entities

### Task

The single entity in this application. Represents one unit of work belonging to the single user.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `id` | UUID | Yes | `gen_random_uuid()` | Unique identifier for the task |
| `title` | TEXT | Yes | — | Task title; must be non-empty |
| `description` | TEXT | No | `NULL` | Optional longer description |
| `completed` | BOOLEAN | Yes | `false` | Whether the task has been marked complete |
| `scheduled_date` | DATE | No | `NULL` | The date the task is scheduled for (no time component); `NULL` means unscheduled |
| `created_at` | TIMESTAMPTZ | Yes | `now()` | Timestamp of creation; used for list ordering (newest first) |

**Validation rules**:
- `title` MUST NOT be empty or whitespace-only. Enforced both client-side (form) and server-side (Server Action).
- `scheduled_date` is a calendar date only (no time). Stored as `DATE` in PostgreSQL.
- `completed` and `scheduled_date` are fully independent; any combination is valid.
- A task with `scheduled_date = NULL` is "unscheduled"; dragging onto a calendar date sets this field.
- A task with `scheduled_date` set remains visible in the left list (with a date badge).

**State transitions**:

```
                    ┌─────────────────────────────────────┐
                    ↓                                     │
[created]  →  [unscheduled, uncompleted]                  │
                    │  ↕ toggle complete                  │ drag to calendar
                    │  [unscheduled, completed]            │
                    │                                     ↓
                    └────────────────────→  [scheduled, uncompleted]
                                                 │  ↕ toggle complete
                                           [scheduled, completed]
                                                 │
                                        drag back to list panel
                                                 ↓
                                       [unscheduled, uncompleted/completed]
```

## Drizzle Schema

```typescript
// db/schema.ts
import { pgTable, uuid, text, boolean, date, timestamp } from 'drizzle-orm/pg-core';
import { sql } from 'drizzle-orm';

export const tasks = pgTable('tasks', {
  id: uuid('id').primaryKey().default(sql`gen_random_uuid()`),
  title: text('title').notNull(),
  description: text('description'),
  completed: boolean('completed').notNull().default(false),
  scheduledDate: date('scheduled_date'),
  createdAt: timestamp('created_at', { withTimezone: true })
    .notNull()
    .default(sql`now()`),
});

export type Task = typeof tasks.$inferSelect;
export type NewTask = typeof tasks.$inferInsert;
```

## Relationships

None. This is a single-table schema. No foreign keys, no joins. Single-user application with no user table.

---

## Indexes

No additional indexes required for v1. The table is expected to hold at most ~200 rows; full scans are negligible. If performance becomes an issue, an index on `scheduled_date` and `created_at` can be added in a future migration.
